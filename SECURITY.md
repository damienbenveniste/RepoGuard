# Security Policy

## Reporting A Vulnerability

Please do not report security vulnerabilities in public issues.

Email Damien Benveniste at `damien.benveniste@gmail.com` with:

- a concise description of the issue;
- steps to reproduce when possible;
- the affected version or commit;
- any known impact or workaround.

I will acknowledge reports as soon as practical and coordinate a fix before
public disclosure when the issue is confirmed.

## Scope

Security-sensitive areas include generated GitHub Actions workflows, filesystem
write behavior, template rendering, command execution, secret handling, and
packaging or publishing configuration.
